# 스냅문서 (SnapDocument) — MVP 기획서 v3

> **버전:** 0.3.0 · **작성일:** 2026-06-30
> **MVP 범위:** PC 웹만. 폰 앱은 제외.

---

## 0. MVP 정의

> **사용자가 정보가 담긴 문서(엑셀·텍스트)를 업로드하면, 미리 등록한 템플릿(엑셀 양식) 형식에 맞춰 자동으로 변환해주는 PC 웹 서비스.**

**MVP 포함:**
- 엑셀 템플릿 등록 (컬럼 자동 분석)
- 정보 문서 업로드 (엑셀 파일 / 텍스트 붙여넣기)
- AI 매핑 (정보 → 템플릿 컬럼에 맞춰 값 채우기)
- 결과 미리보기 + 엑셀 다운로드

**MVP 제외:**
- Flutter 모바일 앱
- OCR / 카메라 촬영
- 결제 / 로그인
- 워드·PPT·한셀 지원

## 1. 사용자 시나리오

### 시나리오 A — 엑셀 데이터를 다른 양식으로 변환

> 김경희는 요양원 행정담당자다. 매달 입소자 정보가 담긴 엑셀(이름, 생년월일, 주소, 보호자, 연락처 등 100행)을 받는데, 이걸 요양원 자체 양식(신청일, 성명, 생년월일, 보호자, 연락처, 주소, 질환, 비고)으로 다시 옮겨야 한다.
>
> 1. `snapdocument.pages.dev` 접속
> 2. [템플릿 등록] 탭 → 요양원 양식 엑셀 올리기 → 컬럼 분석 완료 → "입소상담지"로 저장
> 3. [문서 변환] 탭 → 정보 엑셀 파일 올리기 (입소자 정보 100행)
> 4. 템플릿 "입소상담지" 선택
> 5. 변환 실행 → "이름→성명, 보호자→보호자" 등 컬럼 매핑 미리보기
> 6. 다운로드. 100행이 새 양식으로 들어간 엑셀 완성.

### 시나리오 B — 텍스트 데이터를 엑셀로 정리

> 박준호는 공사현장 소장이다. 매일 카톡으로 "현장: A현장, 작업자: 김철수, 작업내용: 배관, 사용자재: 파이프 20m, 특이사항: 없음" 같은 텍스트가 온다.
>
> 1. PC에서 `snapdocument.pages.dev` 접속
> 2. [문서 변환] 탭 → 텍스트 붙여넣기
> 3. 미리 등록된 "작업일지" 템플릿 선택
> 4. 변환 실행 → 현장명, 작업자, 내용, 자재, 특이사항이 각 컬럼에 매핑
> 5. 다운로드.

### 시나리오 C — 엑셀 파일의 구조를 바꾸기

> A병원에서 받은 엑셀은 "환자명, 진료일, 진료과, 진단명, 처방약, 금액" 순서인데, 보험사 제출용은 "날짜, 이름, 진단, 처방, 비용" 순서다.
>
> 1. 파일 올리기
> 2. 템플릿 선택
> 3. AI가 유사한 컬럼끼리 알아서 매핑
> 4. 미리보기 확인 후 다운로드

## 4. 작업 순서

```
1. 정보 문서 '파일 업로드' 구현 (엑셀 파일 선택)
2. 업로드된 파일 미리보기 (표로 렌더링)
3. 엑셀→엑셀 매핑 로직 (컬럼명 유사도 기반)
4. Worker LLM 연동 (DeepSeek)
5. 기획서 최종 확정 후 코딩
```

## 5. 기술 스택

| 구성 | 기술 |
|------|------|
| PC 웹 | React + TypeScript + Vite |
| 배포 | Cloudflare Pages |
| API | Cloudflare Workers |
| 엑셀 | SheetJS (xlsx) + OOXML 스타일 |
| AI 매핑 | DeepSeek LLM (현재 모의) |
| OCR | 네이버 Clova (Phase 2) |

---

## 6. 불변 원칙

- **데이터는 24시간 후 삭제**
- **템플릿은 사용자가 만든다** (우리가 정해주지 않음)
- **PC에서 모든 게 완결된다** (MVP에선 폰 필요 없음)
- **3단계로 끝낸다**: 정보 올리기 → 템플릿 고르기 → 변환
