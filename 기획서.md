# 스냅문서 (SnapDocument) — 찍으면 문서가 된다

> **버전:** 0.2.0 (MVP 기획 · 극소화)
> **작성일:** 2026-06-30
> **상태:** 협의 중

---

## 1. 프롤로그

책상 위에 영수증 17장, 입소신청서 3부, 건강체크리스트 8장.
다 손으로 엑셀에 옮겨야 한다. 한 장에 3분, 도합 1시간 반.
중간에 오타라도 나면 다시 확인.

"이걸 왜 내가 손으로 치고 있지? 찍으면 알아서 채워주면 안 되나?"

**스냅문서**는 그 질문에서 시작한다.

---

## 2. 제품 개요

### 한 줄

> **종이 문서를 찍으면, 내가 등록한 엑셀 양식에 맞춰 AI가 알아서 채워준다.**

### MVP에서 하는 일 (그리고 안 하는 일)

| ✅ 하는 일 | ❌ 안 하는 일 |
|-----------|-------------|
| 카메라로 종이 촬영 (원근 자동 보정) | 회원가입 / 로그인 |
| 갤러리에서 이미지·스캔본 불러오기 | 결제 |
| 텍스트 직접 붙여넣기 | 사용량 제한 |
| 내가 올린 엑셀 템플릿에 AI 매핑 | 문서 영구 보관 |
| 결과 미리보기 (수정 가능) | 처리 이력 |
| 엑셀(.xlsx) 다운로드 | 워드/PPT 지원 |
| 데이터 24시간 후 자동 삭제 | |

### MVP 구성

| 구성 요소 | 기술 | 하는 일 |
|----------|------|------|
| 모바일 앱 | **Flutter** | 촬영(원근보정)·스캔·텍스트 입력 → 템플릿 선택 → 미리보기 |
| PC 웹 | **React** (단일 페이지) | 템플릿 엑셀 업로드 (1회) |
| 백엔드 | **Cloudflare Workers** | OCR + AI 분석 + 엑셀 생성 |
| 스토리지 | **R2** (24h TTL) | 이미지·결과 임시 저장 |

## 3. 핵심 플로우 (MVP)

```
입력                템플릿 선택         AI 처리            결과
───────────────────────────────────────────────────────────────
📷 카메라 촬영 ─┐
🖼️ 갤러리/스캔 ─┼──→  템플릿 고르기  ──→  OCR+매핑  ──→  미리보기
📝 텍스트 붙이기 ─┘    (등록된 엑셀中)    (DeepSeek)     → 엑셀 다운로드
```

### Step 1 — 템플릿 등록 (PC 웹, 최초 1회만)

```
1. PC에서 snapdocument.pages.dev 접속
2. 엑셀 파일 드래그 (예: 건강체크리스트.xlsx)
3. AI가 컬럼 자동 분석해서 보여줌
4. 이름 붙여서 저장 ("건강체크리스트")
5. 끝. 앱에서 이제 보인다.
```

### Step 2 — 찍고 정리하기 (Flutter 앱, 메인 동작)

```
1. 앱 실행 → 바로 입력 선택 화면
2. [촬영] 탭 → 카메라 열림 → 문서에 맞춰 찍기
   (찍는 순간 자동으로 원근 보정 → 평면 이미지로 변환)
3. 또는 [갤러리] 탭 → 기존 사진/스캔본 선택
   또는 [텍스트] 탭 → 복사한 텍스트 붙여넣기
4. 템플릿 선택 (내가 등록한 것 중에)
5. "분석하기" 탭
6. 몇 초 후 미리보기
7. 값 확인 → 틀린 건 탭해서 수정
8. "완료"
9. PC에서 엑셀 다운로드 (앱에서도 다운로드 링크 복사 가능)
```

## 5. 기술 상세

### 5.1 문서 원근 보정 (Flutter)

촬영 시 자동 처리:

```
1. 카메라 프리뷰에 문서 가이드라인 오버레이
2. 촬영 → OpenCV 또는 ML Kit Document Scanner API로 문서 모서리 감지
3. 4개 모서리 기준으로 원근 변환(Perspective Transform)
4. 정사각형 평면 이미지로 출력
```

Flutter 패키지 후보:
- `flutter_vision` 또는 `google_mlkit_document_scanner` — 내장 문서 스캔
- `edge_detection` — 모서리 감지 + 자동 크롭
- 직접 `dart:ui` Canvas로 warp 처리

### 5.2 OCR — 네이버 Clova OCR

```
POST https://kr1dsx9dj3.apigw.ntruss.com/custom/v1/35469/e036bd81ad7e6c3000dc267b31b9e756e80c1312589e87e8890f64db31829421/general

Headers:
  X-OCR-SECRET: {naver_ocr_secret}
Content-Type: application/json

Body:
{
  "version": "V2",
  "requestId": "{uuid}",
  "timestamp": {unix_ms},
  "images": [
    {
      "format": "jpg",
      "name": "document",
      "data": "{base64}"
    }
  ]
}
```

응답에서 `fields[].inferText` 추출 → 텍스트 하나로 합침.

### 5.3 AI 컬럼 분석 — DeepSeek (템플릿 등록 시)

```
프롬프트 (system):
너는 엑셀 헤더를 분석해서 각 컬럼의 의미와 데이터 타입을 파악하는 도우미다.
반드시 JSON만 출력해라.

프롬프트 (user):
다음 엑셀 헤더를 분석해줘:
날짜 | 성명 | 체온 | 수축기혈압 | 이완기혈압 | 비고

응답 예시:
{"columns":[{"header":"날짜","type":"date","desc":"측정 날짜"},...]}
```

### 5.4 AI 매핑 — DeepSeek (매 분석 시)

```
프롬프트 (system):
너는 OCR로 추출된 텍스트에서, 주어진 컬럼에 맞는 값을 찾아 매핑하는 도우미다.
반드시 JSON만 출력해라. 찾을 수 없는 값은 빈 문자열 ""로.

프롬프트 (user):
OCR 텍스트:
"{ocr_text}"

컬럼 목록:
- 날짜 (date): 측정 날짜
- 성명 (text): 이름
- 체온 (number): 체온(℃)
- 수축기혈압 (number): 수축기 혈압
- 이완기혈압 (number): 이완기 혈압
- 비고 (text): 특이사항

JSON으로 매핑해줘.
```

### 5.5 엑셀 생성

Workers에서 `xlsx` 라이브러리로 템플릿 엑셀을 열고, 매핑된 데이터를 새 행으로 추가한 뒤 R2에 저장.

## 6. 데이터 모델 (MVP · 최소)

```
Device {
  device_id: string (기기식별자, 로그인 없음)
  created_at: datetime
}

Template {
  id: string (PK)
  device_id: string (FK)
  name: string
  file_key: string (R2 원본 경로)
  columns: [{header, type, desc}, ...]
  created_at: datetime
}

Job {
  id: string (PK)
  device_id: string
  template_id: string
  image_key: string (R2 이미지)
  input_type: "camera" | "gallery" | "text"
  ocr_text: string
  mapped_data: JSON
  xlsx_key: string (R2 결과물)
  status: "processing" | "done" | "error"
  created_at: datetime
  expires_at: datetime (24h)
}
```

**기기 식별:** `device_id` = 앱 설치 시 UUID 생성 → 로컬 저장. 서버에 이걸로만 구분.

## 8. MVP 범위

### 포함

- Flutter 앱: 촬영(원근보정)·갤러리·텍스트 입력
- PC 웹: 템플릿 엑셀 업로드 + AI 컬럼 분석
- Workers: OCR → DeepSeek 매핑 → 엑셀 생성
- R2: 24h TTL 자동 삭제
- 기기식별 (로그인 없음)

### 제외 (Phase 2+)

- 회원가입·로그인
- 결제·사용량 제한
- 처리 이력
- 워드·PPT
- 다중 사용자 계정
- 공유·협업
- 다국어

## 10. 불변 원칙

| 원칙 | 이유 |
|------|------|
| **데이터는 24시간 후 자동 삭제** | 신뢰 + 비용 |
| **템플릿은 사용자가 만든다** | 무한 유연성. 우리가 정해주지 않음 |
| **입력 → 템플릿 → 결과, 3단계로 끝** | 기능 크리프 방지 |
| **로그인 없이 쓸 수 있다** | 진입장벽 제로 |
| **원근 보정은 자동** | 사용자가 수동으로 자를 필요 없게 |
| **수정은 틀렸을 때만** | 통과가 기본. 속도 > 완벽 |
| **중간에 안 멈춘다** | 분석 중에도 다른 동작 가능 |
